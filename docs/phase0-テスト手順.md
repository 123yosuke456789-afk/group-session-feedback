# Phase 0 テスト手順（同じ Wi‑Fi で母さんと試す）

## 準備（テスト前に一度だけ）

1. **あなたのパソコン**で、このプロジェクトのフォルダを開く。
2. ターミナルで以下を実行する（まだなら）:
   ```bash
   cd "/Users/yosuke/src/グループセッション"
   npm install
   ```
3. **母さんのスマホ or パソコン**を、**あなたと同じ Wi‑Fi** に接続しておく。

---

## 手順

### ステップ 1：アプリを起動する（あなた）

1. ターミナルで次を実行する:
   ```bash
   cd "/Users/yosuke/src/グループセッション"
   npm run dev
   ```
2. 表示が止まったら起動完了。次のような行が出ているはず:
   - `Local:    http://localhost:3000`
   - **`Network:  http://192.168.x.x:3000`** ← この **Network の URL** をメモする（数字は環境で違う）

3. **このターミナルは閉じない。** 閉じるとアプリが止まる。

---

### ステップ 2：あなたが「セッション」を作る

1. **あなたのパソコンのブラウザ**で、次のどちらかを開く:
   - `http://localhost:3000`
   - または ステップ 1 でメモした `http://192.168.x.x:3000`
2. トップ画面で:
   - **「作品のURL」** に、共有したい作品のURLを入力する。  
     例: 自分のポートフォリオ、Figma の共有リンク、GitHub のページ、画像のURL など。
   - **「セッション名」** は任意（例: 「テスト」）。
3. **「セッションを作成」** をクリックする。
4. フィードバック画面（作品が表示されている画面）に切り替わる。
5. **「招待リンクをコピー」** をクリックする。  
   → クリップボードに `http://192.168.x.x:3000/session/xxxxxxxx` のようなリンクがコピーされる。

---

### ステップ 3：母さんにリンクを渡す

- **招待リンク**（ステップ 2 でコピーしたもの）を、母さんに送る。
  - LINE・メール・メモアプリなどで「このリンクを開いて」と送る。
- もし招待リンクではなく **トップのURL**（`http://192.168.x.x:3000`）を渡した場合は、母さんが開いたあと「セッションに参加」に **あなたが作ったセッションの招待リンク** を貼って「参加する」でもよい（通常は招待リンクをそのまま送るのが簡単）。

**重要:** 母さんが開くのは **あなたのPCのIPのURL** なので、必ず `http://192.168.x.x:3000/...` の形。`localhost` だと母さんの端末では「自分自身」を指してしまうので、**192.168 の方を渡す。**

---

### ステップ 4：母さんにやってもらうこと

1. 送られた **招待リンク** をタップ（またはクリック）して開く。
2. 同じ作品の画面が表示される。
3. **コメントしたい場所** を画面の作品エリアでタップ（またはクリック）する。
4. 右側（または下）に「この位置にコメントを追加」の入力欄が出る。
5. **名前**（任意）と **コメント** を入力して **「送信」** を押す。

---

### ステップ 5：あなた側で確認する

1. あなたのブラウザのフィードバック画面をそのまま開いていると、**約 5 秒以内**に母さんのコメントが反映される（ピンと一覧に表示される）。
2. 作品上の **黄色いピン** をクリックすると、そのコメントが強調表示される。
3. 右側の **コメント一覧** に、母さんの名前（または「匿名」）とコメントが並ぶ。

ここまでできれば **Phase 0 のテスト成功**。

---

## うまくいかないとき

| 症状 | 確認すること |
|------|---------------------|
| 母さんがリンクを開いても「ページを開けません」 | 母さんが **同じ Wi‑Fi** か確認。あなたのPCで **`npm run dev` が動いたまま**か確認。渡したURLが **192.168.x.x** になっているか確認（localhost ではないか）。 |
| 作品が真っ白・表示されない | その作品のURLが **iframe 埋め込みを禁止**している可能性。別のURL（Figma、Google Docs、画像のURLなど）で試す。 |
| コメントが相手に届かない | 5 秒ほど待つ。ブラウザを再読み込みしてみる。 |

---

## テスト後の終了

- テストが終わったら、**ターミナルで `Ctrl + C`** を押して `npm run dev` を止めてよい。
- 次に使うときは、また `npm run dev` からやり直す。
