# リンクは「1つ」で共有する

## 共通のリンクは 1 つだけ

**みんなで使うリンクは 1 つで大丈夫です。**

- **共有用のリンク:** `http://192.168.3.192:3000`（あなたのターミナルに表示されている Network のURL）
- **あなたも母さんも、この同じURLを開く** → どちらも「あなたのパソコンで動いているアプリ」に繋がります。

「あなた用」「母さん用」で別のリンクがあるのではなく、

- **localhost だけ**が「今使っているその端末」を指す特別な名前なので、母さんのパソコンでは使えない
- **192.168.3.192** は「あなたのパソコンの、家のWi‑Fi上での住所」なので、**同じWi‑Fiにいる人なら誰でもこのリンクで同じ画面に繋がる**

という違いがあるだけです。

---

## 母さんのパソコンで 192.168.3.192 を開いても表示されないとき

「接続が拒否されました」「このサイトにアクセスできません」になる主な原因と確認項目です。

### 1. 本当に同じ Wi‑Fiか

- 母さんのパソコンとあなたのパソコンが、**同じWi‑Fi（同じルーター）** に繋がっているか確認する。
- ゲスト用Wi‑Fiや別のSSIDだと、端末同士で通信できないことがある。

### 2. あなたのパソコンでアプリが動いているか

- あなたのターミナルで **`npm run dev` を実行したまま** になっているか。
- ターミナルを閉いたり、止めたりしていないか。

### 3. あなたのパソコンのファイアウォール（Mac）

Mac のファイアウォールが「他の端末からの接続」を止めていることがあります。

**確認・許可の手順（macOS）：**

1. **システム設定**（またはシステム環境設定）を開く
2. **ネットワーク** → **ファイアウォール**（または **セキュリティとプライバシー** → **ファイアウォール**）を開く
3. ファイアウォールが **オン** の場合:
   - **ファイアウォールオプション** を開く
   - **Node** や **node** が一覧にあれば、**接続を許可** にする
   - 一覧にない場合は、いったん **ファイアウォールをオフ** にして、母さんのパソコンからもう一度 `http://192.168.3.192:3000` を開いてみる（テスト用。試したあとでオンに戻してよい）

### 4. ルーターの「AP 隔離」機能

- ルーターの設定に **「AP 隔離」「クライアント分離」** のような項目があると、同じWi‑Fiの端末同士が通信できません。
- ルーターの管理画面でその機能を **オフ** にするか、取扱説明書・プロバイダの案内を確認する。

### 5. あなたのパソコンから母さんのパソコンへ ping（上級者向け）

- 母さんのパソコンのIPアドレスが分かっていれば、あなたのターミナルで  
  `ping 母さんのIP`  
  で届くか試せる（母さん→あなた方向は、母さんのPCで `ping 192.168.3.192` で試せる場合もある）。

---

## まとめ

- **共有するリンクは 1 つ:** `http://192.168.3.192:3000`（と、その下の `/session/xxxxx` などの招待リンク）
- **localhost は「その端末だけ」** なので、母さんには渡さない。
- 母さん側で開けないときは、**同じWi‑Fi・アプリ起動中・あなたのMacのファイアウォール・ルーターのAP隔離** を順に確認する。
